# ADR-002: Engine-Modell-Choice für v0.1.x

## Status
Akzeptiert (3. Mai 2026) — bestätigt durch Verification v0.3.1 (48/48 PASS gegen Spec §1.6 Acceptance-Bands).

## Context
v0.1.x von liver braucht eine pharmakokinetische Engine für BAC-Berechnung. Spec v1.0.0–v1.0.6 sah ethanol-rs (WASM, Rust) mit exponentiellem ka-Absorptions-Modell vor (ka empty=4.0, some=2.5, full=1.5 h⁻¹). Die Implementierung der ersten Code-Pässe hat dieses Modell nicht erreicht:

- **FP1:** Stub statt echter WASM-Anbindung. `--formula widmark` ≡ Watson.
- **FP2:** Wechsel auf pure-TypeScript-Engine, aber `ELIMINATION_RATE = 0.015` als Prozent/h interpretiert (sollte 0.0015 Prozent/h ≡ 0.015 ‰/h). Faktor-10-Bug.
- **FP2.5:** Numerik-Korrektur mit neuen Drifts: `>1s`-Auto-Close, 24h-Window ohne Spec-Anker, Schema-Änderungen.
- **FP3:** β fixed auf 0.0015 Prozent/h, Curve-Fix (`-offset` → `0`), DATABASE_CORRUPTED-Mapping, Concurrency via `BEGIN IMMEDIATE`.
- **FP4:** DATABASE_CORRUPTED-Build-Lücke geschlossen, chrono-Validator gehärtet.
- **FP5:** `formatISOLocal()` TZ-Vorzeichen + Date-Annotation gefixt.

Spec v1.0.8 hat in Cleanup-Pass 8 anerkannt: §1.1 "v0.1.x = TypeScript, v0.2.0 = WASM".

## Decision
**v0.1.x liefert mit pure-TypeScript-Engine in `src/engine/ethanol.ts` aus.**

- Absorptions-Modell: linear über `--duration`. Stomach-Faktoren empty=1.0, some=0.85, full=0.65.
- Eliminations-Rate: β = 0.0015 Prozent/h ≡ 0.015 ‰/h, konstant.
- BAC-Formeln: Watson und Widmark direkt in TypeScript. Watson r m=0.68/f=0.55/o=0.615; Widmark r m=0.73/f=0.66/o=0.695.
- Engine-Boundary: `abv` durch 100 dividieren (Prozent → Fraction), Promille via ×10 aus Prozent.

v0.2.0 plant Migration auf ethanol-rs (WASM) + exponentielles ka. Spec §1.6 Acceptance-Bands sind engine-agnostisch, Migration ist vertragskompatibel.

## Consequences

### Positiv
- Zero-Dependency-Setup. `npm install` reicht.
- Engine-Fehler direkt im TypeScript-Stack-Trace lesbar.
- Acceptance-Bands engine-agnostisch — Migration auf WASM kein Vertragsbruch.
- Schnellere Iterations-Loops, kein WASM-Rebuild.

### Negativ
- Peak-Timing für `--duration 0` liegt bei T0, nicht bei T0+15-25min.
- Curve-Form während Resorption linear, nicht exponentiell-krümmig.
- ka-Werte (4.0/2.5/1.5 h⁻¹) tauchen in v0.1.x nicht auf. Spec §14 als Out-of-Scope.

### Neutral
- Output-Vertrag (§2) unverändert. User-facing JSON identisch.

## Alternatives Considered
- **A: Echtes ethanol-rs WASM in v0.1.x** — verworfen wegen wiederholter Build-/Stub-Drifts FP1–FP2.5.
- **B: Exponentielles ka in pure-TypeScript** — verworfen weil dann keine echte v0.1.x → v0.2.0 Versions-Grenze.
- **C: Hybrid TS + WASM** — verworfen wegen Komplexität.

## Implementation Notes
- Engine lebt komplett in `src/engine/ethanol.ts` (ein File, kein abstraktes Interface — Spec §1.2).
- Watson TBW: m = `2.447 - 0.09516×age + 0.1074×height + 0.3362×weight`; f = `-2.097 + 0.1069×height + 0.2466×weight`; o = Mittelwert. Verteilungs-Volumen = TBW / 0.806.
- Widmark BAC: `(alcohol_g / (weight_kg × r)) - β × hours`.
- Sober-Schwelle: BAC ≤ 0.001 Prozent (≡ 0.01 ‰).

## Migration zu v0.2.0 (nicht normativ)
1. ethanol-rs Crate identifizieren oder bauen.
2. `wasm-pack build --target nodejs --release --features wasm` → `vendor/ethanol-rs/pkg/`.
3. Adapter ruft WASM-Funktionen, behält interne Signatur.
4. Verification v0.4.0 fahren — Bands aus §1.6 müssen PASS bleiben.
5. Spec auf v1.1.x, §1.1 auf "v0.2.0 = WASM live".
